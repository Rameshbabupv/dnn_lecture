# Class Schedule - Day Order Duration Rules

## Standard Class Duration by Day Order

### **Day Order 3 (DO3): ~100 minutes (1hr 40min)**
- Lecture sessions
- Theory-heavy content
- Comprehensive topic coverage
- Time for demonstrations and discussions
- **Typical duration**: 1 hour 40 minutes

### **Day Order 4 (DO4): 40-45 minutes**
- Tutorial/Practical sessions
- Hands-on coding exercises
- Implementation focus
- Student practice time
- **Typical duration**: 40-45 minutes

## Key Implications

### For DO3 (Lecture Sessions):
- Can cover substantial theoretical content
- Time for live coding demonstrations
- Q&A and discussion periods
- Multiple topics or deep dive into complex concepts

### For DO4 (Tutorial Sessions):
- Must be highly focused and practical
- Limited time for theory review
- Emphasis on hands-on implementation
- Students should come prepared
- Quick feedback loops

## Planning Guidelines

### For DO3 Lectures (100 min):
- Plan for 80-90 minutes of active content
- Reserve 10-20 minutes for buffer, Q&A, recap
- Can structure as: 40min + 10min break + 40min + 10min wrap-up

### For DO4 Tutorials (40-45 min):
- Plan for 35-40 minutes of active hands-on work
- 5-10 minutes for setup, intro, and closing
- Students should have prerequisite setup completed
- Provide starter code to maximize coding time
- Concise instructions, minimal theory review

## Important Notes

1. **DO4 sessions are tight** - Every minute counts
2. **Pre-class preparation** is critical for DO4 tutorials
3. **Starter code/templates** help students jump into coding quickly
4. **DO3 sessions** allow for more comprehensive coverage and flexibility
5. **Time management** is crucial - stick to planned duration
